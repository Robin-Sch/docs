## Logging to loki (grafana)

Run:
```bash
docker plugin install grafana/loki-docker-driver:3.3.2-amd64 --alias loki
```

(there might be a newer version, see [here](https://grafana.com/docs/loki/latest/send-data/docker-driver/#install-the-docker-driver-client))

Edit `/etc/docker/daemon.json` and add the following (make sure it's valid JSON):
```json
{
    "log-driver": "loki",
    "log-opts": {
        "loki-url": "http://localhost:3100/loki/api/v1/push",
        "loki-batch-size": "400",
        "loki-retries": "2",
        "loki-max-backoff": "800ms",
        "loki-timeout": "1s",
        "keep-file": "true"
    }
}
```

## Changing subnets

To get a bigger range of subnets I usually set the following in `/etc/docker/daemon.json`:

```json
{
    "ipv6": true,
    "fixed-cidr-v6": "fd0a:9093:73cb:8000::/64",
    "default-address-pools":
    [
        { "base": "100.128.0.0/16", "size": 24 },
        { "base": "fd0a:9093:73cb:8000::/56", "size": 64 }
    ]
}
```

## Blocking IP

To block a particular IP from accessing docker containers, use the following command:

```bash
sudo iptables -I DOCKER-USER 1 --src a.b.c.d -j DROP
```

## Get docker subnets

Use the following command to print all the docker subnets in use:

```bash
docker network inspect $(docker network ls | awk '$3 == "bridge" { print $1}') | jq -r '.[] | .Name + " " + .IPAM.Config[0].Subnet' -
```

## Updating postgresql

```bash
# Make sure only postgres database is running and service is down
docker compose down
docker compose up -d SERVICE-postgres

# Export database
docker exec -it SERVICE-postgres pg_dumpall -U SERVICE > dump.sql
cat dump.sql # verify dump is not empty

# Change version (for upgrade to >=18 make sure volume is set to /var/lib/postgresql, WITHOUT data)
nvim compose.yml

# Delete old/previous database (volume)
docker compose down
docker volume rm SERVICE_postgres_data
docker compose up -d SERVICE-postgres

# Import database
docker exec -i SERVICE-postgres psql -U SERVICE < dump.sql

# Start service and cleanup
docker compose up -d
# if works correctly: rm dump.sql
```
